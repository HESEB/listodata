# V4 Market Intelligence Next Step

## 목적
기존 Dashboard를 단순 뉴스·링크 모음에서 가격, 수급, 질병, 정책, 뉴스가 서로 어떻게 연결되어 시장신호로 이어지는지 보여주는 구조로 확장한다.

## 핵심 개선 방향

### 1. 근거 레이어 추가
`app/data/market_reasoning.json`을 기준 데이터로 사용한다.

각 축종별로 다음 항목을 표시한다.

- 시장신호
- 신뢰도 점수
- 판단 요약
- 근거 연결 흐름
- 보고서형 해석 문장

예시 구조:

```text
한우
가격 상승 압력
+
도축두수 감소 흐름
+
명절·행사 선매입 가능성
=
정육류 강세 요인 우세
```

### 2. 기존 축종 상세 페이지 개선
현재 축종 상세는 `시장 해석`, `핵심 지표`, `기준 데이터 출처`, `관련 자료` 중심이다.

다음 섹션을 추가한다.

```text
AI 판단 근거
- 가격 근거
- 수급 근거
- 질병/방역 근거
- 정책/고시 근거
- 뉴스 근거
```

### 3. 관련 뉴스 연결 강화
뉴스 카드는 단순 최신순이 아니라 해당 축종과 판단 요인에 맞춰 연결한다.

연결 기준:

- 가격: 시세, 지육가, 경락, 도축, 수급
- 수급: 도축량, 도계량, 출하, 사육, 공급
- 질병: 방역, 가축질병, 이동제한, 살처분
- 정책: 고시, 공고, 할당관세, 수급대책
- 소비: 명절, 복날, 휴가철, 외식, 유통

### 4. 화면 구성 제안

축종 상세 페이지 순서:

```text
1. 시장신호 카드
2. AI 판단 근거
3. 가격/수급 핵심지표
4. 기준 데이터 출처
5. 관련 뉴스/자료
6. 보고서 문장
```

### 5. 주의사항

- 확정 표현 금지
- 내부 매입단가, 재고, 계약물량, 벤더 정보 노출 금지
- 뉴스는 보조 근거로만 사용
- 가격과 수급 데이터가 시장신호의 1차 기준

## 반영된 파일

- `app/data/market_reasoning.json`

## 다음 개발 작업

1. `index.html`에서 `market_reasoning.json`을 fetch한다.
2. 축종 상세 페이지의 `detailHTML()`에 판단근거 섹션을 추가한다.
3. 관련 뉴스 필터에 reasoning keyword를 연동한다.
4. Dashboard에 `AI 판단 근거 보기` 섹션을 추가한다.
