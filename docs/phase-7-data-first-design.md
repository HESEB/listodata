# Phase 7-0 — DSS 2.0 데이터 중심 전환 설계

## 1. 목표

현재 HESEB Livestock Terminal을 뉴스 중심 화면에서 **공식 데이터 중심 축산 구매 의사결정 지원 시스템(DSS)**으로 전환한다.

사용자는 Dashboard 진입 후 5초 안에 다음 내용을 파악할 수 있어야 한다.

1. 현재 시장 방향
2. 지금 해야 할 구매 행동
3. 핵심 변화지표
4. 판단 신뢰도와 데이터 커버리지
5. 대표 근거 기사 또는 공식자료

핵심 원칙은 다음과 같다.

```text
공식 데이터가 판단의 주체
뉴스는 판단의 보조 근거
AI는 데이터를 해석하고 문장화
점수는 상세 화면에서만 노출
첫 화면은 방향·행동·핵심 변화 중심
```

## 5. Direction Engine 2.0 설계

### 5.1 입력축

```text
가격       30%
공급       25%
도축/생산  15%
사육       10%
수입/재고  10%
질병        5%
정책/수요   5%
```

축종별 데이터 가용성에 따라 가중치는 재배분할 수 있지만, 가격·공급·생산 데이터가 전체의 60% 이상을 차지해야 한다.

### 5.2 입력 정규화

각 지표 변화율을 -100~+100 신호로 정규화한다.

```text
가격 상승       → 상방 신호
공급 감소       → 상방 신호
도축/생산 감소  → 상방 신호
사육 감소       → 중기 상방 신호
수입 증가       → 하방 신호
재고 증가       → 하방 신호
질병 발생       → 상방 위험 신호
소비 감소       → 하방 신호
```

### 5.3 방향 구간

| 최종 신호 | 출력 |
|---:|---|
| +60 이상 | ▲ 강한 상방 |
| +20~+59 | ▲ 상방 |
| -19~+19 | = 보합 |
| -20~-59 | ▼ 하방 |
| -60 이하 | ▼ 강한 하방 |
| Coverage 60% 미만 | ? 판단 유보 |
| Confidence 55% 미만 | ? 판단 유보 |

### 5.4 신뢰도 계산

```text
Confidence =
공식데이터 비중 35%
데이터 최신성   25%
데이터 커버리지 20%
지표 간 일치도  15%
뉴스 일치도      5%
```

뉴스는 신뢰도의 최대 5%만 담당한다.

## 6. Coverage / Reliability 설계

### 6.1 Coverage

축종별 필수지표 중 정상 확보된 비율이다.

```text
90~100%  매우 높음
75~89%   높음
60~74%   제한적 판단
60% 미만 판단 유보
```

### 6.2 Reliability

데이터별 신뢰도 표시 기준:

```text
★★★★★ 정부·공식기관 원자료, 최신
★★★★☆ 공공기관·협회 자료, 최신
★★★☆☆ 언론 또는 2차 가공자료
★★☆☆☆ 기업자료·추정치
★☆☆☆☆ 출처 불명 또는 오래된 자료
```

Dashboard에는 별점만 표시하고 상세 화면에서 산식과 출처를 공개한다.

## 7. Recommendation Engine 기본 규칙

Recommendation Engine은 방향만으로 추천하지 않고 재고·계약·가격·신뢰도를 함께 본다.

### 7.1 기본 출력

```text
비축 검토
추가 계약
고정가 협의
관망
재고 소진
행사 운영
대체 원료 검토
판단 유보
```

### 7.2 초기 규칙

| 조건 | 추천행동 |
|---|---|
| 상방 + Confidence 75% 이상 + 재고 낮음 | 비축 검토 |
| 강한 상방 + 계약물량 부족 | 추가 계약 |
| 상방 + 가격 변동성 높음 | 고정가 협의 |
| 보합 + 재고 적정 | 관망 |
| 하방 + 재고 높음 | 재고 소진 |
| 하방 + 수요 회복 가능 | 행사 운영 |
| 특정 축종 강한 상방 + 대체축종 보합/하방 | 대체 원료 검토 |
| Coverage 또는 Confidence 기준 미달 | 판단 유보 |

실제 재고·계약 데이터가 연결되기 전까지는 `시장 데이터 기준 추천`으로 명시한다.

축종 적합도  15점

### 8.2 제외 기준

## 9. Report Sentence Engine 설계

문장은 아래 순서로 생성한다.

```text
핵심 공식 데이터 변화
→ 공급/수요 의미 해석
→ 시장 방향
→ 대표 근거
→ 구매 검토 행동
```

예시:

```text
도축두수는 전년 대비 감소한 반면 경락가격은 전월 대비 상승해 공급 부담에 따른 가격 강세가 이어지는 모습입니다. 최근 공식자료와 대표 기사에서도 동일한 흐름이 확인되고 있어 단기 상방 가능성이 높으며, 운영 재고가 낮은 품목은 추가 비축을 검토할 필요가 있습니다.
```

금지 원칙:

```text
근거 없는 확정 표현 금지
오래된 뉴스의 현재형 사용 금지
데이터가 부족한 경우 상방/하방 강제 산출 금지
수치 없는 뉴스 나열형 문장 금지
```
