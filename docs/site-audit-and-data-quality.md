# 사이트 점검 및 데이터 품질 기준

## 점검 결과

현재 사이트 기능은 크게 두 영역으로 분리되어 있다.

```text
v3.3.0 영역
- 뉴스
- 자료실
- 정책/질병/산업/신제품 필터
- 자료출처
- 리포트 생성

v4.1 영역
- 숫자 기반 핵심지표
- 시장신호 점수
- 점수 산식
- 자동수집 파이프라인
```

## 발견된 문제

### 1. 샘플 데이터가 실제 데이터처럼 보이는 문제
`market_series_sample.json`의 값이 `market_metrics.json`으로 계산되어 화면에 표시되면서 사용자가 실제 공식 데이터로 오해할 수 있었다.

### 2. 공식수집과 수동입력 구분 부족
KREI, 양계협회, KAHIS 등 공식 출처와 연결된 데이터인지, 단순 샘플/수동 스냅샷인지 표시가 부족했다.

### 3. 수집 실패 상태 표시 부족
어댑터가 실패하거나 값을 찾지 못한 경우에도 화면에서 명확히 드러나지 않았다.

## 이번 수정 기준

### 데이터 상태 필드 추가
`market_metrics.json`의 축종 및 개별 지표에 아래 필드를 추가한다.

```json
{
  "data_status": "OFFICIAL_FETCHED | SAMPLE | FETCH_FAILED | SAMPLE_OR_PARTIAL",
  "data_status_label": "공식수집 | 샘플/수동 | 수집실패 | 샘플/부분연동"
}
```

### 화면 표시 기준

- 공식 자동수집 성공값: `공식수집`
- 수동/샘플 스냅샷: `샘플/수동`
- 일부만 공식수집: `샘플/부분연동`
- 어댑터 실패: `수집실패`

## 실제 데이터 반영 우선순위

### 1순위: 대한양계협회 닭 지육가
이미 HTML 어댑터가 추가되어 있다.

```text
scripts/adapters/chicken_price_adapter.py
```

다음 확인 필요:

- GitHub Actions 실행 후 `chicken_price_snapshot.json` 생성 여부
- 추출 숫자의 정확도
- 9~10호 컬럼 정확 매칭 여부

### 2순위: KREI OASIS 돼지 지육가
OASIS 페이지에 CSV다운로드 UI는 존재하나, 실제 다운로드 API 파라미터 확인이 필요하다.

### 3순위: KREI OASIS 한우 지육가
돼지 지육가와 동일 구조로 어댑터 확장 가능성이 높다.

### 4순위: KREI OASIS 도축·도계량
월별 확정치와 잠정치 구분 필요.

### 5순위: KAHIS / 농식품부 질병자료
뉴스/목록 기반이므로 가격 지표가 아니라 질병 영향도 점수의 보조 지표로 연결한다.

## 운영 원칙

- 실제 공식 출처에서 수집 성공한 값만 `공식수집`으로 표시한다.
- 샘플값은 절대 실제 데이터처럼 표시하지 않는다.
- 수집 실패 시 기존 데이터를 덮어쓰지 않는다.
- 화면에는 데이터 상태를 반드시 표시한다.
- 내부 매입단가, 재고, 계약물량, 벤더 정보는 수집/표시하지 않는다.
